\section{Implementation}
In Figure~\ref{fig:bernstein_class_diagram} is shown the class diagram of the Bernstein library. This first set of classes represent the model domanin ofthe system.
\begin{itemize}
    \item \textbf{BernsteinPolynomial} \\
        It represent the whole polynomial and provides the main API to manipulate it.
    \item \textbf{BernsteinBasis} \\
        Is the interface that represent the basis of the polynomial. It contain the whole logic to compute the basis and provides the hook (i.e., the \texttt{clamp()} method) that allow its implementation to define their type of basis.
    \item \textbf{LinearBernsteinBasis} \\
        Is the classic Bernstein basis. The input \texttt{x} is mapped into $[0,1]$, by deafult, or in the $[a,b]$ intervall if it specified.
    \item \textbf{ExponentialBernsteinBasis} \\
        Is the exponential Bernstein basis. The input \texttt{x} is mapped into $t=1-e^{-\lambda x}$.
\end{itemize}

\subsection{Bernstein Polynomial Calculus}
To impleent the calculus of the Bernstein polynomial we have considered two different approaches: the direct evaluation (the choosen one) and de Casteljau's algorithm.
\subsubsection{Direct Evaluation}
With the direct evaluation we compute the value of the polynomial using the explicit formula $P(x) = \sum_{i=0}^{n} f(\tfrac{i}{n}) \cdot b_{i,n}(x)$, where $f(\tfrac{i}{n})$ is the $i$-th coefficient of degree $n$ and $b_{i,n}(x)$ is the $i$-th basis function of degree $n$.

Using this technique we have the following consequences:
\begin{itemize}
    \item \textbf{Advantage}: It's very simple to implement. If the coefficients are precalculated and stored in a table, the evaluation can be done in $O(n)$ time.
    \item \textbf{Disadvantage}: It can be numerically unstable for high degrees (i.e., already for $n>20$) due to the large coefficients and small powers.
\end{itemize}

\subsubsection{de Casteljau's algorithm}
It's a recursive approach similar to the Horner's scheme for standard polynomials: to calulate the value of the polynomial of given degree we use the value of the polynomial of degree $n-1$.

Using this technique we have the following consequences:
\begin{itemize}
    \item \textbf{Advantage}: The recursion is efficient if implemented in in iterative way. It is very stable also for high degrees.
    \item \textbf{Disadvantage}: It has a higher compotational cost of $O(n^2)$ due to the double nested loop: if the degree is very high, it can be significantly slower than direct evaluation. It is usefull if we are considering only the classic linear basis.
\end{itemize}

\subsection{Testing}
The testing of the Bernstein library considers three different aspects: the basis, the polynomial evalutatio and the Bernstein statical interface.

\subsubsection{Basis Testing}
We have tested the main properties and situatin:
\begin{itemize}
    \item The correct result for known and small degree in specific points.
    \item The validation of the unity partition property.
    \item The correct handling of invalid input.
    \item Verified that basis functions are always non-negative within their support.
\end{itemize}

\subsubsection{Polynomial Calculus}
We tested the following aspects:
\begin{itemize}
    \item If we have all coefficients $\beta_i = C$, then $P(x) = C$.
    \item For a linear basis on $[0,1]$, if $\beta_i = i/n$, then $P(x) = x$.
    \item The boundary conditions for which $P(x_{min}) = \beta_0$ and $P(x_{max}) = \beta_n$.
    \item Verification of the symmetry property $P(x; \beta) = P(1-x; \text{reverse}(\beta))$ for the standard basis.
\end{itemize}

\subsubsection{Optimization}
To optimize the performance of the librery we have used some tricks:
\begin{itemize}
    \item The class responsable of calculating the binamial coefficient uses a cache to store the previously calculated values. Since a lot of the same coefficients are used during the evaluation of the basis functions, this can significantly improve the performance.
    \item During the calculation of the polynomial we calculate the mapped value $t$ (i.e., \texttt{t=basis.clamp(x)}) of the input $x$ only one time; we also calculate the basis value directly in the loop, without calling the \texttt{eval()} method of the basis.
\end{itemize}

This optimization brought a speedup about $6\times$: before the optimization the evalution of a Bernstein polynomial of degree 1000 took about 68ms; after the optimization it takes about 11ms; so we have $speed-up=\tfrac{68}{11}=6.18\times$.
